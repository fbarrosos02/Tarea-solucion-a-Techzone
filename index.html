Planificación y Cronograma de Trabajo

Funcionalidad Duración estimada
Catálogo de productos2 semanas
Carrito de compras1 semana
Registro e inicio de sesión2 semanas
Historial de pedidos1 semana
Proceso de pago2 semanas
Notificaciones por correo1 semana
Seguimiento de pedidos1 semana

Este cronograma distribuye el trabajo a lo largo de los 3 meses de desarrollo, con tiempos dedicados a cada uno de los requisitos esenciales.

Definición de Requisitos

Requisitos Funcionales
RequisitoDescripción
Catálogo de productosLos usuarios podrán ver todos los productos organizados en diferentes categorías.
Carrito de comprasLos usuarios podrán añadir productos a un carrito de compras.
Registro e inicio de sesiónLos usuarios podrán registrarse y gestionar su cuenta.

Diseño de la Arquitectura

  • Frontend: Se desarrollará con React para una interfaz rápida y adaptable.
  • Backend: Node.js gestionará operaciones de usuarios, carritos y pagos.
  • Base de datos: MongoDB será usada para manejar productos y pedidos.

Implementación

  • Configurar la API REST para productos y pagos.
  • Desarrollar las interfaces de usuario con React.
  • Integrar pasarelas de pago.

Pruebas

  • Pruebas de carga con JMeter.
  • Pruebas de compatibilidad en distintos dispositivos.

Despliegue y Mantenimiento

  • La aplicación será desplegada en AWS.
  • Monitorización de rendimiento con CloudWatch.
